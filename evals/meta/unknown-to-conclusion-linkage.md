# Eval: Unknown-to-Conclusion Linkage

Use this eval when a report appears uncertainty-aware but still overstates what the final judgment, ranking, timing, or recommendation can support.

## Goal

Distinguish between:

1. uncertainty that is merely disclosed
2. uncertainty that actually constrains the conclusion

A report should not get credit simply for having an uncertainty section. The key question is whether unresolved unknowns visibly changed what the memo was willing to conclude.

---

## Typical failure patterns

- caveats appear, but the headline recommendation remains equally strong
- the report admits important missing evidence, but keeps precise quantitative language
- a ranking is presented as stable even though one unresolved variable could plausibly flip it
- the memo identifies a promising direction, but still speaks as if a full-strength go decision were justified
- unknowns are listed in a separate section without affecting action intensity, scope, timing, or confidence

---

## What this eval is testing

### Failure Mode 1: Cosmetic uncertainty

The report names gaps, but they do not change the conclusion.

Examples:
- says data is limited, but still gives a strong winner
- notes unclear commercial traction, but still writes as if scale is confirmed
- says cost assumptions are rough, but still gives precise ROI or payback guidance

### Failure Mode 2: Insufficient downgrade

The report weakens wording slightly, but not enough for the size of the unresolved unknown.

Examples:
- says `likely` or `appears` while still keeping an over-precise recommendation
- says ranking may change, but does not explain what variable would change it
- calls a recommendation `conditional`, but still recommends full commitment rather than pilot / wait / scoped action

### Failure Mode 3: Correct directional view, wrong precision level

The report may be right about direction, but wrong about how much precision or confidence it claims.

Examples:
- directional positive view may be justified, but target timing is not
- shortlist inclusion may be justified, but top-rank certainty is not
- low-regret pilot may be justified, but full rollout is not

---

## Pass criteria

A good answer should:

1. make visible what still can be concluded
2. make visible what can no longer be concluded because of remaining unknowns
3. downgrade recommendation strength, ranking stability, timing confidence, or precision when required
4. name reversal conditions when a realistic unknown could flip the answer
5. distinguish directional judgment from precise or commitment-grade judgment

---

## Scoring guide

Use a simple 0-2 scale.

### 0 = cosmetic uncertainty only
- the report discloses gaps, but they do not materially constrain the conclusion

### 1 = partial downgrade
- the report narrows some language, but the final conclusion is still stronger or more precise than the evidence supports

### 2 = real conclusion constraint
- unresolved unknowns visibly change the scope, precision, confidence, ranking strength, timing confidence, or recommended action

---

## Review questions

When using this eval, ask:

- Which unknowns are actually load-bearing?
- What part of the conclusion should those unknowns constrain?
- Did the report downgrade that specific part?
- Is the memo still claiming a stable winner where only a conditional lead is justified?
- Is the memo still claiming commitment-grade action where only low-regret action is justified?
- Did the report name what would reverse, weaken, or delay the conclusion?

---

## Output format for reviewers

When you apply this eval, summarize the result as:

- **Load-bearing unknowns:**
- **What the report still can conclude:**
- **What the report should no longer conclude:**
- **Visible downgrade behavior:**
- **Where the downgrade is still too weak:**
- **Diagnosis:** cosmetic uncertainty / insufficient downgrade / precision overreach
- **Best next fix:** template hardening / checklist hardening / source-discipline hardening / new route-specific audit

---

## Why this eval exists

As the skill improves, more weak reports will sound disciplined because they now mention uncertainty. This eval exists to prevent uncertainty disclosure from becoming a stylistic checkbox instead of a real constraint on judgment.
