# Eval Asset Index

This index tracks the active case-level eval assets in `evals/cases/`.

It is intentionally lightweight: it does not replace the case files, the failure taxonomy, or the system map. It exists so maintainers can quickly scan route coverage, recurring failure families, primary disciplines, and the rule/checklist target each case protects.

## Status Legend

- `active` — normalized case that belongs in the regression/audit asset set.
- `candidate` — useful failure material that still needs normalization before it becomes an active eval.
- `needs-review` — tracked case whose route, failure family, or current-rule verdict should be rechecked.
- `stale` — case expresses an old contract and should be rewritten or retired.
- `superseded` — case is covered by a newer, clearer eval.

## Current Rule Status Legend

- `pass` — current rules should treat this pattern as acceptable.
- `fail` — current rules should reject this pattern as a hard fail.
- `warn` — current rules should treat this pattern as conditional or requiring correction.
- `manual-review` — current rules require reviewer judgment or context-specific classification.
