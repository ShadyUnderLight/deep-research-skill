# Eval: 寒武纪 Competitive Positioning Near-Pass Case

## Goal

Test whether a Competitive Positioning / First-tier report with strong structural execution (scope/metric/timeframe, dimension-level conclusions, overall-label gate, evidence-strength separation) can still achieve only Conditional Pass when secondary route hard-fail verification is not itemized by check and quantitative role labels are not fully per-checklist.

This is the **first Competitive Positioning route eval case** across all 40+ cases.

## Real case pattern

A user-provided report "寒武纪是否进入全球AI芯片第一梯队" (Has Cambricon Entered the Global AI Chip First Tier?) dated **2026-06-05** demonstrates this pattern.

**What was done well:**
- ✅ Competitive Positioning route correctly selected with explicit scope/metric/timeframe
- ✅ Dimension-level conclusions (D1-D6) with separate evidence-strength assessment
- ✅ Overall-label gate explicitly run (§7) — refuses to compress into a single global tier label
- ✅ Structured source register with ID, type, date, URL, reliability, claims supported
- ✅ Body-level `[Sxx]` citations present throughout
- ✅ Strong counter-evidence (§8) with per-point refutation
- ✅ Uncertainty register with monitoring signals and reversal conditions
- ✅ No `第一梯队` loose usage — separates global/China/non-Huawei-domestic/procurement/investment contexts
- ✅ No prestige-label collapse — explicitly refuses a merged tier label
- ✅ Judgment-first opening with conditional framing

**Core issues (conditional pass level):**
- ❌ **Secondary route hard-fail verification not itemized** — §12 declares listed-company + technical-deep-dive secondary hard-fails "satisfied" without per-condition verification
- ❌ **[IN]/[UN] format not per-checklist** — uses `[Sxx]` with "推断" table instead of `[IN]` format; acceptable but not full pass
- ❌ **Table-level quantitative role labels incomplete** — §6 has fact/inference/unknown separation but many table rows lack observed/estimate/vendor-claim/analyst-inference markers
- ❌ **Self-assessment slightly overconfident** — §12 states "均通过" when secondary route and quantitative labeling are conditional

## Scoring

- **Full Pass**: secondary route hard-fail itemized + full quantitative role labels + honest self-assessment
- **Conditional Pass**: strong primary route execution but secondary verification + label format gaps (this case's level, close to Pass)
- **Fail**: primary route hard-fail triggered

## Key signal

This is a **near-Pass** report — the strongest among recent cases. It suggests the Round 4 #182 (Source Register template) and #183 (quantitative role columns) fixes are starting to take effect for the primary route, but secondary route verification and cross-format consistency still lag.
