# Current-State Verification

Use this file when the task involves products, model versions, device lineups, pricing, packaging, release cycles, company status, leadership, rankings, policy changes, or any other fast-moving topic.

Do not assume model memory is current. Verify the current state before analysis.

## Core rule

When a conclusion depends on what is true now, confirm the current snapshot first.

Do not rely on:

- prior model knowledge
- stale summaries
- old reviews or evergreen SEO pages
- historical authority when newer primary sources exist

## Topics that require a current-state check

Always perform explicit current-state verification for:

- consumer electronics
- AI models and versioned software
- SaaS pricing and plans
- company leadership, fundraising, layoffs, M&A
- current product lineup or release generation
- rankings, market-share claims, and trend snapshots
- regulations or policy proposals that may have changed recently

## Current-state checklist

Verify as many of these as relevant before synthesis:

- latest product generation or version
- current product lineup
- current pricing or packaging
- current company positioning
- most recent major announcement or release
- whether newer information supersedes older sources
- date context for the sources used

## Source priority

Prefer sources in this order:

1. official current product/company pages
2. official announcements, press releases, release notes
3. current docs, changelogs, support pages
4. reputable reporting after checking primary sources
5. older summaries only as background

## Freshness-first search patterns

Use searches like:

- `[topic] official`
- `[topic] latest`
- `[topic] current`
- `[topic] current lineup`
- `[topic] release date`
- `[topic] pricing`
- `site:<official-domain> [topic]`

Examples:

- `site:mi.com Xiaomi 17`
- `Xiaomi official phone lineup`
- `Claude pricing official`
- `site:openai.com GPT-5 current pricing`

## Common failure modes

Avoid these mistakes:

- treating an old review as current status
- letting broad background pages override newer product pages
- mixing multiple generations without checking dates
- assuming a lineup or plan stayed unchanged
- using press summaries without checking the original announcement

## How to handle uncertainty

If the current state cannot be verified clearly:

- say that directly
- reduce confidence
- avoid filling the gap with likely-but-stale knowledge
- separate historical context from verified current state

## Output expectation

For fast-moving topics, make the current-state check visible in the final answer when useful.

Examples:

- `As of the latest official lineup page reviewed, ...`
- `Current pricing could not be fully verified from official sources.`
- `Older summaries mention X, but newer primary sources indicate Y.`
